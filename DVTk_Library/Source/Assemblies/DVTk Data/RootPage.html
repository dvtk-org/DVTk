Dicom Validation ToolKit (DVTK)

Introduction

The Dicom Validation ToolKit is a SDK framework to perform DICOM validation tests.

It consists of .NET components, see Architectural View Of The Design Model

Capabilities Of DVTK

DVTK;
  • is a software utility that assists in testing DICOM conformance,
  • provides an independant measurement of the accuracy of the Product‘s DICOM interface according to the DICOM Standard,
  • provides an independant measurement of the accuracy of the Product‘s DICOM interface according to the Product‘s Conf. Statement,
  • can be used as either SCU or SCP with a direct connection to the Product (via TCP/IP),
  • measures accuracy in terms of syntax and semantic correctness,
DVTK does not;
  • perform inter-operability testing
  • check what a specific medical device does with the transferred data.
  • replace human interpretation
Human interpretation will always remain necessary!

Architectural View Of The Design Model

The DVTK SDK consists of several components. These components are provided in the form of .Net assemblies.

The DVTK and DvtkData assemblies may be used by any integrating application. Examples of integration applications;

  • GUI applications
  • Test frameworks such as N-Unit
  • Script environments (see also DVTK SDK ScriptSupport)
DVTK provides DICOM connectivity and validation of DICOM connectivity with a System Under Test (SUT).

DICOM and DUL Messages are defined with the classes defined in DvtkData. These messages are send and received within DVTK sessions. Results are typically safed to XML results output files.

Architectural View Of The Deployment Model

This diagram shows the physical connection between the DVTK hosting system (Dicom Validation Tool System) and the System Under Test. These systems are connected by means of a TCP/IP. The DICOM communication between these systems is subject to the DVTK validation algorithms.

With DVTK it is also possible to validate DICOM media files stored on a file-system.
